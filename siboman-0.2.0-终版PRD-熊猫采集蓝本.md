## 1. 实机对标结论（本轮新增证据）

### 1.1 熊猫采集箱 = siboman 采集模块的蓝本

来源截图：`screenshots/v3/xiongmao/01_list_top.png` ~ `07_publish_shop_select.png`

**熊猫采集箱的 4 个核心设计（siboman 必须抄）**：

#### ① 批量粘贴入口（`03_paste_dialog.png`）
```
按钮：+ 新增1688/淘宝/拼多多采集
弹窗：
  * 采集链接（必填）
  [        大文本域，一行一个 URL       ]
   填写说明：
    1. 每行输入一个商品详情页 URL，支持批量输入；
    2. 请勿输入空行或重复链接，后台会按行依次执行采集；
    3. 支持最多输入 5 个采集链接，超出部分将被忽略。
  [确定] [取消]
```

> **关键克制**：熊猫没做花哨的"SKU 粘贴 / 自动匹配开关"。**siboman 不要过度设计**，v1 就做"URL 一行一个 + 上限 5 条"。

#### ② 编辑弹窗顶部（`04_edit_top.png`）
```
标题：商品属性编辑
一行：商品类目 [下拉+搜索]   [🖼一键加水印] [🌐翻译所有图片] [🔤翻译标题] [✨AI 智能填充]
       灰字：支持输入关键词搜索类目                    [☑ 解析 SKU 尺寸]
必填信息：
  * 商品名称 [Браслет на ногу медный...]     * 卖家 ID [8367755177]
  * 税率 VAT [0% (免增值税)]
分割线：商品必填属性 —— 从这里开始动态渲染
```

**关键设计原则**：
- **AI 工具栏是编辑弹窗的一等公民**（4 个按钮一排在顶栏，不在每个字段旁）
- **类目切换 → 属性区重新加载**（"切换类目将重新加载属性"提示已给出）
- **卖家 ID = offer_id**，默认自动生成，可覆盖

#### ③ 编辑弹窗中部 SKU 变体表格（`05_edit_middle.png`）
```
「其他信息」标题下：
SKU 列表                     [利润计算] [Nano Banana AI 生图]
┌───┬────────┬──────────────────────┬─────────┬─────────────────────┬────┐
│ # │ 颜色    │ 主图 (最多 4 张)       │ 标题     │ 变体可变属性         │操作 │
│   │ 样本图  │ 每张：翻译/替换/3:4/删│         │ 件数 / 商品颜色     │    │
├───┼────────┼──────────────────────┼─────────┼─────────────────────┼────┤
│ 1 │ [图]    │ [图][图][图][图]      │серебри..│ [___] 银色 ▼        │ 🗑  │
└───┴────────┴──────────────────────┴─────────┴─────────────────────┴────┘
```

**关键设计**：
- SKU 是**表格式**编辑（不是每个 SKU 单独一个 Modal）
- 每个变体行 4 张主图，每张图有 4 个小操作（翻译文字 / 替换 / 3:4 裁剪 / 删除）
- 变体可变属性由类目决定（这里是"件数 + 颜色"）
- 「利润计算」按钮右侧固定，点开弹小面板

#### ④ 编辑弹窗底部动态属性（`06_edit_bottom_v2.png`）
从 Ozon 类目属性动态渲染的字段（此案例是"女士卫生巾"品类）：
- 卖家代码 [文本]
- 签名 18+ [Radio: 是/否]
- 原厂包装数量 [数字加减器]
- 卫生巾的种类 [下拉：日/夜]  ← 灰字"从列表中选择，您只能指定一个值"
- 月经内裤大小 [下拉：S/M]
- 欧亚经济联盟 HS 编码 [长下拉]
- 组合成类似的产品 [文本]
- #主题标签 [多标签文本]
- 统一计量单位商品数量 [数字加减器]
- 封面类型 [下拉]

**每个字段下都有小灰字**说明约束（"只能选一个"、"必须从列表选"等）—— siboman 必须复现这个体验。

#### ⑤ 编辑弹窗底栏（关键克制点）
```
      [关 闭]  [保存修改]
```
**熊猫不在编辑弹窗内直接上架**！编辑完只能"保存修改"，回列表再点行操作触发送入上架流程。

#### ⑥ 送入上架弹窗（`07_publish_shop_select.png`）
```
标题：选择店铺
  ○ threelatte              ← Radio 单选（本用户只有一家）
  
  上传备注：可记录货源地址、采购平台、采购价格等信息（选填）
  [                                     0/1500 ]
  
      [取消]  [确定]
```

**关键设计**：
- **上传备注框**：这是**采购溯源**的关键，用户可以在这里记录"从某某 1688 店铺 采购价 12 元"，将来订单发生时可以回溯
- 单选 radio（本用户只有 1 店），**siboman 支持多店时要改成"每店独立配置"或单选**

### 1.2 MyERP 订单模块 = siboman 订单模块的蓝本

来源截图：`screenshots/v3/myerp-orders/01_order_top.png` ~ `06_batch_label.png`

**MyERP 订单模块的关键设计（数据实测）**：

#### ① 订单列表列（`01_order_top.png`）
实测 8 列：
- ☑️ 复选框
- 倒计时（如 "99 小时"）
- 货件 / 状态（含 posting_number + 复制按钮 + 状态徽章 + 时间 + 子状态"已登记"）
- **店铺**（Three Latte 徽章）← 多店铺环境的关键列
- 商品（缩略图 + 俄文标题 + SKU + 数量）
- 仓库 / 配送
- 订单金额
- 利润
- 操作

顶部 Tab：`所有订单(34) | 等待备货(0) | 等待发运(3) | 已超时(0) | 运输中(25) | 有争议的(0) | 已签收(0) | 已取消(6)`

顶部工具栏：`[当前店铺 ▼]  [货件编号 ▼]  [🔍 搜索货件号/SKU...]  [📥 订单导出 ▼]`

#### ② 订单详情右滑抽屉（`03_detail_top.png`）
**这是 MyERP 最亮眼的设计**——右滑抽屉 + 4 个 Tab：


- 批量备货
- 批量打印标签（面单）

## 2. siboman 0.2.0 采集模块 PRD（**参考熊猫**）

### 2.1 三层结构（严格按熊猫）

```
第一层：采集箱列表页  (/products/collect)
   ↓ 点行[编辑]
第二层：编辑弹窗       (商品属性编辑 Modal)
   ↓ 保存修改，回列表
   ↓ 列表行点[送入上架]（或"重新上传"）
第三层：店铺选择弹窗   (Modal，含备注)
   ↓ 确定 → 提交上架任务，进入队列
```

**siboman v0.1 现状 vs v0.2 目标**：

| 层 | v0.1 | v0.2（按熊猫做） |
|---|-----|----------------|
| L1 列表 | ✅ 已有 | 顶部加「+ 新增采集」批量粘贴按钮；行操作从"送入上架"扩展为 `[编辑][送入上架][删除]` |
| L2 编辑 | ❌ 无 | **新建**，Modal 弹窗形态，宽度 1200px |
| L3 店铺 | ❌ 无 | **新建**，Modal 弹窗，含店铺选择 + 采购备注框 |

### 2.2 L1 · 采集箱列表页

#### 2.2.1 字段来源与转换逻辑

| 列 | 字段来源 | 转换/展示逻辑 |
|---|--------|-------------|
| ☑️ | 前端 | 全选/半选 |
| 采集链接 | `collect_items.source_url` | 超长省略；hover 显示完整；点击新窗口打开 |
| 商品图 | `collect_items.main_image` | 60x60 圆角；hover 放大 400x400 |
| 采集状态 | `collect_items.collect_status` | 徽章：pending(黄) / scraping(蓝) / scraped(绿) / failed(红) |
| 商品标题 | `collect_items.title_ru`(主) / `title_zh`(副) | 双行显示，超出省略 |
| 类目 | `collect_items.category_path_zh` | "美容和卫生 / 个人卫生用品 / 女士卫生巾" |
| 发布状态 | `collect_items.publish_status` | 徽章：未上传(灰) / 上传中(蓝) / 上传成功(绿) / 上传失败(红) |
| 采集时间 | `collect_items.created_at` | YYYY-MM-DD |
| 操作 | — | `[编辑][送入上架/重新上传][删除]` |

#### 2.2.2 顶部工具栏

```
┌─────────────────────────────────────────────────────────────────┐
│ 标题：[请输入商品标题______________]  [搜索]                     │

```

- 「+ 新增采集」按钮 = 打开 §2.2.3 批量粘贴弹窗
- 「刷新」= 拉取采集端最新状态

#### 2.2.3 批量粘贴弹窗（严格按熊猫 `03_paste_dialog.png`）

```
标题：添加商品采集任务
* 采集链接 [
   请输入采集链接，一行一个 URL
   
   
]

ℹ️ 填写说明：
  1. 每行输入一个需要采集的商品详情页 URL，支持批量输入；
  2. 请勿输入空行或重复链接，后台会按行依次执行采集；
  3. 支持最多输入 5 个采集链接，超出部分将被忽略。

                                       [确定]  [取消]
```

**AC**：
- **AC-CB01**：粘贴文本自动按 `\n` 切割，去空行、去重复
- **AC-CB02**：超过 5 条截断并提示 "已忽略后 N 条"
- **AC-CB03**：不校验 URL 有效性（交给采集端），只校验非空
- **AC-CB04**：确定 → 后端插 N 条 collect_items 记录（status=pending），触发 collector 异步抓取；弹窗关闭；列表 3 秒轮询直到全部抓完

## 5. 多店铺方案（TL 明确要求的第 3 个 AC）

### 5.1 全站店铺切换器 UI

**位置**：所有列表页顶部工具栏第一位（订单/商品/采集箱/库存都统一）。

**形态**：
```
[Three Latte ▼]     ← 单选下拉，可搜索
   ├─ 全部店铺
   ├─ Three Latte  ✓（当前）
   ├─ Eight Middle
   ├─ Polarwind
   ├─ Xiami Official Store
   └─ ...
```

**特殊：采集箱送入上架**（§2.4）是**多选**，其他所有筛选场景都是**单选或全部**。

### 5.2 后端 store_id 传递规范

#### 5.2.1 全局约定

- 前端全局状态：`window.__CURRENT_STORE_ID__`（Vue 里存在 Pinia store）
- 切换店铺时：
  1. 更新全局状态
  2. 触发全局事件 `store-changed`
  3. 所有列表页监听此事件 → 自动刷新数据
  4. URL query 加 `?store_id=xxx`（便于分享/回访）

#### 5.2.2 API 契约

**所有列表接口必须支持 `store_id` 参数**：

```
GET /api/seller/products?store_id=xxx&status=selling&last_id=...
GET /api/seller/orders?store_id=xxx&tab=awaiting_packaging&last_id=...
GET /api/collect/list?store_id=xxx&status=pending
```

**特殊值**：
- `store_id=all` → 返回该用户所有店铺数据
- 不传 store_id → 默认使用 user_default_store（存 app_users.default_store_id）

#### 5.2.3 数据模型改动

```sql
-- 现有表加 store_id
ALTER TABLE collect_items ADD COLUMN store_id BIGINT REFERENCES app_stores(id);
ALTER TABLE app_products ADD COLUMN store_id BIGINT REFERENCES app_stores(id);
ALTER TABLE order_notes ADD COLUMN store_id BIGINT REFERENCES app_stores(id);

-- 索引
CREATE INDEX idx_products_store_status ON app_products(store_id, ozon_state);
CREATE INDEX idx_orders_store_status ON orders(store_id, status);
```

#### 5.2.4 多店铺同发（采集箱→上架）

```
用户在采集箱[送入上架]勾选 3 家店
   ↓
POST /api/collect/:collect_id/publish
Body: {
  store_ids: [1, 2, 3],
  remark: "1688 张三店，采购价 ¥12"
}
   ↓
后端：
  for each store_id in store_ids:
    创建一条 listing_jobs 记录 { collect_id, store_id, remark, status: 'pending' }
    异步调 Ozon /v3/products/import (用该店的 credentials)
    成功后：
      insert app_products (offer_id, store_id, purchase_remark, ...)
```

| 场景 | UI 反馈 |
|-----|-------|
| 顶部切换店铺 | 全站导航栏底部出现 1s 蓝色进度条 |

## 8. 组件规范（防止"改一片崩全站"）

**完整规范见** [siboman-0.2.0-全功能定义与组件规范.md](siboman-0.2.0-全功能定义与组件规范.md) §4。

**本次强化**：

1. **编辑 Modal 组件（`<CollectEditModal>`）** 必须支持配置化：
   ```js
   props = {
     mode: 'collect' | 'product',   // 采集箱 vs 商品列表
     itemId,
     onSave  // 由业务侧决定调哪个 API
   }
   ```
   商品列表和采集箱**共用**这一个组件，未来任何字段调整只改一处。

2. **店铺切换器 (`<StoreSwitcher>`)** 抽成全局组件，所有列表页复用。

3. **动态属性表单 (`<DynamicAttributeForm>`)** —— 根据后端返回的 `{type, options, required, description}` 自动渲染，前端不用为每个类目改代码。

## 9. 后端 API 契约（配套）

### 9.1 采集箱

| Method | 路径 | 说明 |
|--------|------|-----|
| GET | `/api/collect/list?store_id=&status=&last_id=` | 列表（Cursor 分页） |
| POST | `/api/collect/batch-add` | 批量粘贴 URL（body: `{ urls: [], store_id }`） |
| GET | `/api/collect/:id/edit-data` | 打开编辑 Modal 时拉全字段 |
| PUT | `/api/collect/:id/draft` | 保存草稿（body: 全部字段） |
| POST | `/api/collect/:id/publish` | 送入上架（body: `{ store_ids, remark }`） |
| DELETE | `/api/collect/:id` | 删除 |

### 9.2 商品

| Method | 路径 | 说明 |
|--------|------|-----|
| GET | `/api/seller/products?store_id=&tab=&search=&last_id=` | 列表 |
| GET | `/api/seller/products/:offer_id/edit-data` | 编辑 Modal 数据 |
| PUT | `/api/seller/products/:offer_id` | 保存并同步 Ozon |
| PATCH | `/api/seller/products/:offer_id/field` | 行内编辑单字段（`{key: 'price', value: 1200}`） |
| POST | `/api/seller/products/batch-price` | 批量改价 |
| POST | `/api/seller/products/batch-stock` | 批量改库存 |

### 9.3 订单

| Method | 路径 | 说明 |
|--------|------|-----|
| GET | `/api/seller/orders?store_id=&tab=&search=&last_id=` | 列表 |
| GET | `/api/seller/orders/:posting_number` | 详情（完整 4 Tab 数据） |
| POST | `/api/seller/orders/batch-ship-package` | 批量备货（body: `{ posting_numbers: [] }`） |
| POST | `/api/seller/orders/batch-labels` | 批量面单（body: `{ posting_numbers: [] }`；返回 `{ job_id }` 后 poll 状态） |
| GET | `/api/seller/orders/labels/:job_id/download` | 下载合并 PDF |

### 9.4 类目/属性

| Method | 路径 | 说明 |
|--------|------|-----|
| GET | `/api/ozon/categories/tree` | 类目树（缓存 24h） |
| GET | `/api/ozon/categories/:cat_id/attributes` | 类目属性 schema |
| GET | `/api/ozon/brands?q=` | 品牌搜索 |

## 10. AI 工具后端契约

| 按钮 | 后端 API | 输入 | 输出 |
