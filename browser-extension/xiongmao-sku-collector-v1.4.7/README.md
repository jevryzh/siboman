# 熊猫 Ozon 数据采集器

这是一个本地浏览器插件，用来采集熊猫上架、MY 选品页以及 Ozon Seller 仓库订单页里的表格数据。

支持页面：

- `system.xiongmaoshangjia.cn`
- `my.jizhangerp.com`

## 安装

1. 打开 Chrome 或 Edge。
2. 地址栏输入 `chrome://extensions/` 并回车。
3. 打开右上角的「开发者模式」。
4. 点击「加载已解压的扩展程序」。
5. 选择这个文件夹：

   `xiongmao-sku-collector`

如果之前已经加载过旧版插件，在扩展程序页面点击这个插件卡片上的刷新按钮，让浏览器读取最新的脚本。

## 使用

1. 打开熊猫上架热度排行、MY 榜单选品页，或仓库订单页（如"来自我的仓库的订单"）。
2. 点击浏览器右上角的「采集页面数据」图标。
3. 点击「采集当前页」。
4. 采集完成后，可以复制 CSV，也可以下载 CSV/JSON。

## 采集内容

### 仓库订单页（如"来自我的仓库的订单"）

插件会自动识别订单表，按列采集：

- 货件编号
- 状态
- 已接收
- 发送日期
- 不逾期
- 照片（商品图链接）
- 数量（例如 `1个`）
- 货号（例如 `NSDJB-QIANLAN`、`2759948396-g86h`）
- 商品名称（俄文/中文标题）
- 价格（带 ₽/￥/¥ 符号）
- 仓库（例如 `CEL陆运`）
- 配送服务（例如 `CEL Economy Extra Small`）
- 方式（详细配送方式说明）

每个货件编号作为唯一键，自动去重。

v1.4.7 修复了 Ozon Seller 后台订单页的几个问题：

- 保留 `28 Jun 22:00`、`4 Jul 至07:00` 里的小时分钟
- `截止发运时间` 使用 Ozon 页面里的发运截止时间，不再错误顺延 7 天
- 模板 A-W 列保留，同时在后面追加 `数量`、`货号`、`Ozon商品名称`、`Ozon状态`、`Ozon仓库`、`Ozon价格`
- 商品名称列同时保留货号和 Ozon 商品名，避免只导出货号

### 选品页（商品信息）

插件会把"商品信息"里的内容拆成：

- 商品名称
- SKU
- 商品编号
- 品牌
- 商品链接
- 图片链接

同时会尽量读取页面表格里的每一列，例如：

- 类目
- 所属类目（自动拆成"主类目 / 子类目"）
- 机会判断
- 月销量
- 月销售额
- 均价
- 月销售额环比
- 月浏览量
- 日均销量
- 日均销售额

如果页面使用懒加载，保持「自动向下采集当前列表」打开，插件会向下滚动当前列表并合并已加载的行。

如果页面有分页，需要翻到下一页后再点一次「采集当前页」。
