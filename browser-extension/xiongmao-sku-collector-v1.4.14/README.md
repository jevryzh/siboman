# 熊猫 Ozon 数据采集器

这是一个本地浏览器插件，用来采集熊猫上架、MY 选品页以及 Ozon Seller 仓库订单页里的表格数据。

支持页面：

- `system.xiongmaoshangjia.cn`
- `my.jizhangerp.com`

## 安装

1. 打开 Chrome 或 Edge。
2. 地址栏输入 `chrome://extensions/` 并回车。
3. 打开右上角的「开发者模式」。
4. 点击「加载已解压的扩展程序」。
5. 选择这个文件夹：

   `xiongmao-sku-collector`

如果之前已经加载过旧版插件，在扩展程序页面点击这个插件卡片上的刷新按钮，让浏览器读取最新的脚本。

## 使用

1. 打开熊猫上架热度排行、MY 榜单选品页，或仓库订单页（如"来自我的仓库的订单"）。
2. 点击浏览器右上角的「采集页面数据」图标。
3. 点击「采集当前页」。
4. 采集完成后，可以复制 CSV，也可以下载 CSV/JSON。

## 带图导出

- 「复制到腾讯文档（内嵌小图）」：插件会先把商品图下载下来，压缩成 96px 左右的小图，再把 `data:image` 图片直接写入剪贴板 HTML，减少腾讯文档抓取外链失败的问题。
- 「下载 Excel（内嵌图）」：生成 `.xlsx`，图片会真正写进 Excel 文件内部，这是最稳的离线带图方式。
- 「下载 HTML（内嵌图）」：生成一个可离线打开的 HTML，图片使用 base64 内嵌，浏览器打开能看到图。
- 「下载 CSV」：CSV 格式本身不能保存图片，只能保存链接或 `=IMAGE()` 公式；是否显示图片取决于 WPS/Excel 是否支持公式、能否访问图片链接。
- 「下载 ZIP」：包含 CSV 和 `images/` 文件夹，适合归档；CSV 里是图片路径，不等于把图片嵌进 CSV。

v1.4.14 修复：

- 移除 Chrome 不接受的非法权限 `https://ir-*.ozonusercontent.com/*`
- 补充实际 Ozon 图片域名 `https://*.ozonru.cn/*`
- 图片下载改为不带 cookie 的请求，修复 Ozon CDN `Access-Control-Allow-Origin: *` 与 `credentials: include` 冲突导致的 CORS 失败
- 支持从 `currentSrc`、`srcset`、`data-src` 识别真实图片地址
- HTML 下载改用更稳定的图片下载路径
- Excel 内嵌图片导出会把图片写进 `.xlsx` 内部的 `xl/media/`
- 腾讯文档粘贴改为本地内嵌压缩小图，不再依赖公共图床或 Ozon 原图外链
- Excel 的“商品图片”单元格不再写纯 URL，改为 `=IMAGE("图片地址")` 公式，同时保留内嵌图片对象；从 Excel 复制行到腾讯文档时，不会再只是裸图片地址

注意：Excel/WPS 里的图片本质上是浮动对象。逐行复制到网页表格时，不同软件对图片对象支持不一致；最稳的方式仍然是用插件里的「复制到腾讯文档（内嵌小图）」或直接把 `.xlsx` 导入腾讯文档。

## 采集内容

### 仓库订单页（如"来自我的仓库的订单"）

插件会自动识别订单表，按列采集：

- 货件编号
- 状态
- 已接收
- 发送日期
- 不逾期
- 照片（商品图链接）
- 数量（例如 `1个`）
- 货号（例如 `NSDJB-QIANLAN`、`2759948396-g86h`）
- 商品名称（俄文/中文标题）
- 价格（带 ₽/￥/¥ 符号）
- 仓库（例如 `CEL陆运`）
- 配送服务（例如 `CEL Economy Extra Small`）
- 方式（详细配送方式说明）

每个货件编号作为唯一键，自动去重。

### 选品页（商品信息）

插件会把"商品信息"里的内容拆成：

- 商品名称
- SKU
- 商品编号
- 品牌
- 商品链接
- 图片链接

同时会尽量读取页面表格里的每一列，例如：

- 类目
- 所属类目（自动拆成"主类目 / 子类目"）
- 机会判断
- 月销量
- 月销售额
- 均价
- 月销售额环比
- 月浏览量
- 日均销量
- 日均销售额

如果页面使用懒加载，保持「自动向下采集当前列表」打开，插件会向下滚动当前列表并合并已加载的行。

如果页面有分页，需要翻到下一页后再点一次「采集当前页」。
