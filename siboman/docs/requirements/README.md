## 1. 背景与目标

逐梦 ERP 正处于从早期的 "Ozon-1688 找货工具" 向 "全功能跨境 ERP" 进化的关键阶段。基于对 MyERP 的深度调研，本项目旨在填补商品精细化管理、订单履约自动化以及数据决策支持方面的空白。

本轮需求的核心目标是：**提效、保命、增长**。
- **提效**：通过采集箱、AI 改图、批量上架及发货管理，大幅缩短从选品到出单的作业时间。
- **保命**：通过库存管理、利润趋势分析及成本拆解，确保每单利润透明，规避经营风险。
- **增长**：通过类目分析、榜单选品，解决"卖什么"的战略难题，实现业绩突破。

## 3. 优先级说明

- **P0 = 本季度必做**（1~2 个月）：核心链路功能，直接影响日产出。
- **P1 = 本季度尽量做**（3~4 个月）：提升运营深度和稳定性的功能。
- **P2 = 有余力再做**：锦上添花或探索性功能。

## 4. 建议交付顺序

## 5. 共用约定

### 5.1 术语表
- **Ozon**：俄罗斯领先的电商平台 (https://www.ozon.ru)。
- **1688**：阿里巴巴旗下 B2B 采供平台。
- **FBS / FBO**：Ozon 的履约模式。FBS 为卖家自发货（本项目重点），FBO 为平台代发货。
- **采集端 (Collector)**：运行在本地（如 Mac）的 Playwright 爬虫程序，负责高强度网页抓取。
- **MiniMax-M3 / image-01**：本项目采用的国产大模型，分别负责文本处理和图像生成。
- **SPA**：Single Page Application，本项目前端基于原生 JS 实现的单页架构。
